# Theme CSS strategy

`azamat-ui-kit` does not import global CSS from the package entry.

## Why

The consuming app should own:

- Tailwind imports
- dark/light mode strategy
- CSS variables
- theme presets
- font imports
- global body/html styles

This keeps the UI kit reusable across Kassa, LMS, Restaurant and future dashboard projects without forcing one global stylesheet into every app.

## Recommended setup

Run:

```bash
npx azamat-ui-kit init
```

During init, choose the app global CSS file, usually:

```txt
src/index.css
```

The CLI appends a marked block:

```css
/* azamat-ui-kit theme start */
:root { /* light variables */ }
.dark { /* dark variables */ }
/* azamat-ui-kit theme end */
```

The marker prevents duplicate writes on repeated init.

## What to keep in the app CSS

For Tailwind v4 apps, your app CSS should usually keep its own imports first:

```css
@import "tailwindcss";
@import "tw-animate-css";
@import "shadcn/tailwind.css";

/* azamat-ui-kit theme start */
/* generated by init */
/* azamat-ui-kit theme end */
```

## Dark mode

Dark mode is class-based. Toggle `.dark` on the root element:

```html
<html class="dark">
```

or from React:

```ts
document.documentElement.classList.toggle("dark", isDark)
```

## Component package rule

Do not import package CSS manually:

```ts
// not needed
// import "azamat-ui-kit/style.css"
```

Just import components:

```tsx
import { Button, DataTable, ModalShell } from "azamat-ui-kit"
```
